Documentación Técnica

Guías de referencia y arquitecturas de servicios Microsoft — desde la ingesta de datos hasta el consumo analítico.

Herramientas Azure para Gestión de Datos

Clasificación de servicios según su rol en la Arquitectura Medallion: desde la ingesta de datos crudos hasta el consumo analítico.

BRONCE — Datos Crudos → PLATA — Datos Limpios → ORO — Datos Listos
BRONCE — Ingesta y Almacenamiento Crudo
Captura y almacena datos tal como llegan, sin transformación
Azure Data Factory
Orquestador ETL

Extrae datos automáticamente desde múltiples fuentes (bases de datos, APIs, archivos) y los mueve hacia el Data Lake. Es el punto de entrada principal de datos.

Azure Event Hubs / IoT Hub
Ingesta en Tiempo Real

Captura datos de alta frecuencia en tiempo real: sensores, telemetría, streams. Ideal para datos que llegan continuamente como oleaje, viento o movimiento de naves.

Azure Data Lake Storage Gen2
Repositorio Principal

Almacén centralizado de todos los datos: crudos (Bronce) y semi-procesados (Plata). Gran escala, bajo costo, compatible con Spark y Hadoop.

PLATA — Procesamiento y Transformación
Limpia, transforma y estandariza los datos crudos
Azure Databricks
Motor de Análisis y ETL

Plataforma avanzada de análisis basada en Apache Spark. Limpia datos, aplica reglas de calidad, hace transformaciones complejas y prepara datasets para modelos de Machine Learning.

Azure Stream Analytics
Procesamiento Streaming

Procesa datos en tiempo real con latencia de milisegundos. Detecta patrones al vuelo, dispara alertas automáticas y transforma streams de sensores antes de almacenarlos.

ORO — Modelado y Consumo
Datos listos para análisis, reportes y modelos predictivos
Azure Synapse Analytics
Data Warehouse + Big Data

Plataforma unificada que combina Data Warehouse (SQL) con Big Data (Spark). Aquí se construyen los modelos analíticos y Data Marts listos para consumo por Power BI o Machine Learning.

Azure Machine Learning
Modelos Predictivos e IA

Gestiona el ciclo completo de ML: entrenamiento, versionado y despliegue de modelos. Expone los modelos como endpoints (APIs) para que otras aplicaciones los consuman.

Power BI
Visualización y Reportes

Herramienta de Business Intelligence que consume datos de la capa Oro para generar dashboards, reportes automáticos e indicadores operacionales para las áreas del negocio.

TRANSVERSAL — Gobernanza, Seguridad y Automatización
Servicios que operan en todas las capas simultáneamente
Microsoft Purview
Gobernanza de Datos

Catálogo central de datos: qué existe, dónde está, quién lo usa. Gestiona el linaje (trazabilidad de origen a destino), calidad y clasificación por sensibilidad.

Azure Key Vault
Seguridad

Almacena de forma segura credenciales, contraseñas, claves de cifrado y certificados. Ningún servicio guarda sus propias contraseñas: todas vienen de aquí.

Azure Monitor
Monitoreo

Observabilidad de toda la plataforma: métricas, logs y alertas de rendimiento. Supervisa pipelines, APIs y recursos de cómputo en tiempo real.

Azure Logic Apps
Automatización Low-Code

Crea flujos de trabajo automáticos sin código: enviar alertas por Teams/email, disparar acciones cuando un modelo detecta una anomalía, notificar a operaciones.

Azure Functions
Cómputo Serverless

Ejecuta pequeñas piezas de código bajo demanda sin gestionar servidores. Ideal para auditorías, notificaciones automáticas y micro-tareas disparadas por eventos.

Azure API Management
Gestión de APIs

Puerta de entrada controlada para exponer modelos y datos a aplicaciones internas o externas. Gestiona seguridad, límites de uso (throttling) y documentación de APIs.

Azure Active Directory (Entra ID)
Identidad y Acceso

Controla quién puede acceder a qué: usuarios, roles y permisos sobre Power BI, APIs, Data Lake y modelos. Es la capa de identidad de toda la plataforma.

Azure Container Apps
Despliegue de Modelos

Ejecuta microservicios y modelos de ML empaquetados como contenedores. Es el backend operacional que expone las predicciones como APIs listas para consumir.

Azure RBAC — Jerarquía de Ámbitos

Haz clic en cada nivel del árbol para ver su alcance de acceso. Los roles asignados en un nivel se heredan hacia abajo.

Estructura de Jerarquía
Detalle del Ámbito
Tenant (Inquilino raíz)
Alcance del Rol
El nivel más alto de toda la jerarquía. Es el inquilino de Microsoft Entra ID que contiene todo — identidades, suscripciones y recursos de la organización.
Ejemplo
contoso.onmicrosoft.com es el tenant. Todo lo que existe en Azure de Contoso vive dentro de este tenant.
No se asignan roles RBAC directamente al tenant. Se usa el Grupo de Administración raíz para eso.
La herencia fluye hacia abajo, nunca hacia los lados
Grupo de Administración Raíz
Alcance del Rol
Existe uno solo por tenant. Un rol asignado aquí se hereda por TODAS las suscripciones y recursos del tenant sin excepción.
Ejemplo
Asignar Lector aquí = puede ver absolutamente todos los recursos de todas las suscripciones del tenant.
Usar con extrema precaución — el acceso se propaga a todo el tenant.
La herencia fluye hacia abajo, nunca hacia los lados
Grupos de Administración Anidados
Alcance del Rol
Permiten organizar suscripciones en jerarquías. Un rol asignado aquí se hereda por todas las suscripciones y recursos dentro de ese grupo.
Ejemplo
Grupo Admin "América Latina" contiene suscripciones de Colombia, México y Argentina. Contribuyente aquí = accede a las tres.
Máximo 6 niveles de anidamiento y 10,000 grupos por tenant.
La herencia fluye hacia abajo, nunca hacia los lados
Sub-grupo de Administración
Alcance del Rol
Los grupos de administración pueden anidarse hasta 6 niveles. Cada sub-grupo hereda las políticas y roles del grupo padre, y los propaga hacia abajo.
Ejemplo de jerarquía real
Contoso Global → América Latina → Colombia → Producción Colombia → Suscripción Prod-COL
Útil para empresas multinacionales con estructuras complejas por región o unidad de negocio.
La herencia fluye hacia abajo, nunca hacia los lados
Suscripción
Alcance del Rol
Acceso a todos los grupos de recursos y recursos dentro de ESTA suscripción únicamente. Otras suscripciones del tenant no se ven afectadas.
Ejemplo
Asignar Lector aquí = puede ver todo en esta suscripción, pero NO tiene acceso a otras suscripciones del mismo tenant.
La herencia fluye hacia abajo, nunca hacia los lados
Grupo de Recursos
Alcance del Rol
Acceso únicamente a los recursos contenidos en este grupo. Otros grupos de la misma suscripción no se ven afectados.
Ejemplo
Asignar Contribuyente en "RG-Marketing" = gestiona VMs y Storage de ese grupo, sin tocar "RG-Finanzas" ni otros grupos.
La herencia fluye hacia abajo, nunca hacia los lados
Recurso Individual
Alcance del Rol
Acceso únicamente a ESTE recurso específico. Es el nivel más granular y el que mejor cumple el principio de privilegios mínimos.
Ejemplo
Asignar Colaborador SMB en "fileshare2" = solo accede a ese fileshare, no a otros recursos de la misma cuenta de almacenamiento.
Nivel recomendado cuando se aplica el principio de privilegios mínimos.
Nivel más bajo — no hay más herencia debajo
