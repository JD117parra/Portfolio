Azure En evaluación

Migración de intranet corporativa on-premises a Azure

El problema

Empresa de 170 funcionarios con intranet corporativa operando sobre WordPress + MySQL en contenedores Docker sobre servidor físico propio (8 núcleos, 32 GB RAM, 12 TB). Sin respaldo estructurado, sin plan de recuperación ante desastres y con todo el peso operativo sobre el equipo de TI interno.

La solución

Evaluación comparativa IaaS vs PaaS: Azure VM frente a Azure Container Apps + Azure Database for MySQL Flexible Server. Para el tamaño de este workload, la VM resultó más costo-efectiva. Arquitectura propuesta: Azure VM para WordPress, Azure Database for MySQL Flexible Server y Azure Backup con respaldos diarios automatizados y retención de 30 días.

Resultados proyectados

  • Reducción estimada del 40–60% en costos operativos vs operación on-premises
  • Almacenamiento ajustado de 12 TB sobredimensionado a 1–3 TB según uso real
  • SLA de 99.9% de disponibilidad garantizado por Microsoft
  • RPO de 24 horas y RTO de 1–4 horas con Azure Backup
  • Eliminación completa de carga de mantenimiento físico del servidor
Azure En evaluación

Migración de sistema legacy Visual Basic 6.0 + SQL Server 2016 a Azure

El problema

Sistema legacy crítico compuesto por aplicación Visual Basic 6.0 (descontinuada desde 2008) y SQL Server 2016 Standard con 4 bases de datos (~40 GB). El soporte extendido de SQL Server 2016 termina en julio 2026, generando riesgo de seguridad y cumplimiento. Sin opciones de VPS robustas y con urgencia de migrar antes del fin de soporte.

La solución

Arquitectura IaaS completa: Azure VM D4s_v5 con Windows Server 2025 Datacenter y SQL Server 2019/2022 en la misma VM. Se descartó PaaS por incompatibilidad con VB6 y posible uso de SQL Agent, Linked Servers y Database Mail. Estrategia "lift-and-shift" sin reescribir código ni modificar el driver MSOLEDBSQL19. Complementado con Managed Disk Premium SSD 256 GB, Azure Backup y VNet privada con NSG.

Resultados esperados

  • Eliminación del riesgo por fin de soporte SQL Server 2016 antes de julio 2026
  • SLA de 99.9% de disponibilidad garantizado por Microsoft
  • Ahorro del 40–55% en licenciamiento con Azure Hybrid Benefit
  • Costo estimado: ~$415–540 USD/mes con AHB vs ~$815–940 sin AHB
  • Migración sin modificar código ni drivers — mínimo riesgo operativo

Microsoft 365 Propuesta técnica en curso

Propuesta de licenciamiento Microsoft 365 Enterprise para entidad financiera regulada

El problema

Entidad fiduciaria del sector financiero regulado en Colombia, sin un stack de productividad y seguridad estructurado bajo Microsoft 365. La exposición era triple: riesgo de incumplimiento normativo, protección insuficiente de datos sensibles de clientes, y falta de trazabilidad para auditorías.

La solución

Propuesta de licenciamiento M365 en modelo híbrido por roles: M365 E3 como base para la mayoría de usuarios (DLP, retención, SharePoint, Teams, Exchange Online 100 GB) con Defender for Office 365 Plan 1 como add-on obligatorio. M365 E5 exclusivamente para roles críticos — Compliance, Legal, Riesgos y Ejecutivos — que requieren eDiscovery avanzado, Defender Plan 2 y cumplimiento avanzado. Se descartaron explícitamente F1 y Business Standard por insuficiencia para el sector.

Resultados

  • Recomendación de licenciamiento estructurada y justificada técnicamente para presentar al cliente
  • Modelo híbrido E3/E5 optimizado por costo-beneficio según rol, sin sobrelicenciar ni sublicenciar
  • Cobertura obligatoria de Defender for Office 365 Plan 1 ante ataques de correo en sector financiero
  • Alineación con requisitos de cumplimiento regulatorio del sector financiero colombiano
Microsoft 365 Preparación para reunión técnica

Auditoría y consolidación de licenciamiento Microsoft 365 — cliente con SKUs redundantes

El problema

Cliente del sector minero en Chile con ~379 licencias activas bajo NCE y 7 SKUs distintos acumulados de forma no planificada. Se identificaron al menos tres fuentes de gasto innecesario: 84 licencias de Exchange Online Plan 1 standalone duplicando buzones ya incluidos en Business Basic, 12 licencias de Apps for Business redundantes, y 65 licencias de Defender for Business standalone incluidas en Business Premium. Dos bloques de vencimiento (junio y octubre 2026) indicaban crecimiento orgánico sin estructura.

La solución

Consultoría técnica de optimización de licenciamiento para preparar la reunión de reestructuración con el cliente. Mapeo completo de SKUs activos, detección de redundancias y propuesta de consolidación en tres fases: Discovery, Optimización y Roadmap. La consolidación se planificó alineada con las fechas de renovación NCE — las reducciones de licencias solo son posibles en renovación anual, no antes. Se mantienen los 20 usuarios con Exchange Plan 2 por requisitos de Litigation Hold y retención legal.

Resultados

  • Identificación de al menos 3 fuentes de gasto innecesario en el stack de licenciamiento
  • Hoja de ruta de consolidación alineada con vencimientos NCE de junio y octubre 2026
  • Propuesta técnica lista para presentar al cliente con justificación por cada cambio recomendado
  • Exchange Plan 2 preservado para los 20 usuarios con necesidades de compliance y retención legal

Power Platform Propuesta técnica en curso

Automatización del proceso de egreso universitario con Power Automate

El problema

Institución universitaria del sector privado con un proceso de egreso (emisión de Analítico Final y Diploma) gestionado de forma manual y desarticulada entre tres áreas internas y el sistema ministerial SIDCer. Sin trazabilidad entre etapas, sin notificaciones automáticas al alumno y sin control del SLA de 3 meses ni escalamiento ante vencimientos.

La solución

Arquitectura de automatización con Power Automate como orquestador central, sin reemplazar los sistemas existentes. Cuatro flujos especializados por responsabilidad: recepción del trámite vía trigger HTTP desde el portal institucional, monitoreo de cambios de estado en el sistema de gestión, control del SLA con recordatorios programados a los 30, 60, 75 y 85 días, y cierre con entrega digital al alumno. Integración vía API REST como método preferido, con SQL Server como alternativa. Licencia Process por flujo y cuenta de servicio centralizada como dueña de los flujos.

Resultados proyectados

  • Eliminación del seguimiento manual entre áreas con notificación automática al alumno en cada cambio de estado
  • Trazabilidad completa del trámite con historial de estados, timestamps y responsables por etapa
  • Control automatizado del SLA de 3 meses con escalamiento proactivo antes del vencimiento
  • Modelo replicable a otros trámites institucionales sin rediseñar la infraestructura existente
Power Platform Propuesta técnica en curso

Automatización del proceso de facturación con Power Automate y Office Scripts

El problema

PYME del sector servicios con un equipo de facturación que exportaba datos manualmente desde su sistema de gestión hacia Excel, construyendo tablas cliente por cliente. El proceso tomaba entre 20 y 30 minutos por cliente, con errores frecuentes que generaban retrasos en la emisión de facturas.

La solución

Power Automate como orquestador del flujo: detecta automáticamente la llegada de nuevos archivos exportados en SharePoint/OneDrive y dispara la ejecución de un Office Script (TypeScript) que limpia, normaliza y estructura el archivo Excel. Al finalizar, envía una notificación automática por email. Todo el stack corre sobre Microsoft 365 Business Standard existente, sin licencias adicionales.

Resultados proyectados

  • Eliminación del procesamiento manual de 20 a 30 minutos por cliente
  • Reducción de errores en tablas de facturación por normalización automática
  • Costo adicional cero — solución cubierta por Microsoft 365 Business Standard
  • Tiempo estimado de implementación: 3 a 4 días

Arquitectura Microsoft

Nube privada con licenciamiento SPLA para proveedor MSP

El problema

Empresa de tecnología que buscaba ofrecer servicios administrados (MSP) a clientes terceros bajo modelo SPLA. Sin un modelo de licenciamiento definido, enfrentaba riesgo de incumplimiento al hospedar SQL Server y Windows Server sin el programa adecuado, sin claridad sobre si licenciar por servidor físico, por VM o por usuario.

La solución

Implementación del programa SPLA como modelo de licenciamiento mensual. Windows Server Datacenter licenciado por cores físicos en los hosts (cubre VMs ilimitadas). SQL Server Enterprise licenciado por vCores por VM. VMware vSphere con reglas DRS Affinity para fijar las VMs de SQL a los hosts licenciados — demostrable en auditoría y sin necesidad de licenciar todo el cluster.

Resultados estimados

  • Cumplimiento legal total bajo programa SPLA — sin riesgo de auditoría
  • Windows Server Datacenter cubre VMs ilimitadas por host con una sola licencia del host físico
  • Costo mensual estimado ~$5,180 USD recuperable con margen comercial sobre clientes finales
  • Modelo OPEX sin inversión inicial en licencias perpetuas
Arquitectura Microsoft Pre-venta

Arquitectura híbrida para simulador profesional marítimo

El problema

Una asociación profesional del sector marítimo en Argentina adquirió un simulador profesional de Maniobra, Navegación y Pesca que requiere hardware físico específico. El cliente consultó si era viable migrar la solución 100% a Azure para evitar la compra de hardware Lenovo ThinkStation P2 con GPUs NVIDIA RTX serie 50.

La solución

Arquitectura híbrida: 10 equipos físicos Lenovo on-premises (9 estaciones Windows 11 Pro + 1 servidor coordinador Windows Server), licenciados vía CSP, complementados con Azure Backup, Microsoft Intune y Microsoft 365. La migración 100% a nube fue descartada: las GPUs RTX 5060/5070 Ti no existen en ningún datacenter cloud, el simulador requiere salida de video a 3 pantallas físicas de 65" y la arquitectura del software exige latencia sub-milisegundo en red local.

Estimados referenciales

  • Costo estimado del servidor en Azure (si se valida): ~$90–100 USD/mes en Brazil South con 40 horas de uso mensual
  • Gestión remota de los 10 equipos físicos desde cualquier ubicación vía Intune
  • Protección del servidor on-premises con Azure Backup sin reemplazar el almacenamiento local
  • Licenciamiento CSP: el canal más ágil para clientes en Argentina sin EA o MPSA previo
